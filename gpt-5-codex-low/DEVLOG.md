# Development Log

## Setup

Created a new isolated run directory named `gpt-5-codex` and placed all implementation files inside it. The app is a minimal Vite + React project with no chess-specific dependencies.

## Architecture

The implementation separates the game engine from the React UI:

- `src/chessEngine.js` owns board state, legal move generation, move application, check detection, checkmate, stalemate, castling rights, en passant state, captured pieces, and promotion.
- `src/App.jsx` owns interaction state, selected squares, promotion dialog state, localStorage persistence, and rendering.
- `src/styles.css` defines the responsive layout, board, move indicators, captured pieces, and modal presentation.

Board state is represented as an 8x8 array of piece objects shaped as `{ color, type }`. Game state also includes `turn`, `castling`, `enPassant`, `captured`, `status`, and recent move `history`.

## Rule Handling

Pseudo-legal moves are generated by piece type, then filtered by applying each move to a cloned state and rejecting positions where the moving side's king remains in or moves into check. This keeps pinned pieces, discovered checks, king moves, and check evasions governed by one king-safety path.

Castling is modeled as a king move with metadata. The engine verifies that the king and rook still have castling rights, the path is empty, the king is not currently in check, and the king does not pass through or land on an attacked square. Castling rights are removed when a king moves, when a rook moves from its home square, or when a rook is captured on its home square.

En passant stores the skipped square after a two-square pawn advance. Opposing pawns may capture into that square on the immediately following move, and the captured pawn is removed from its adjacent square.

Promotion is marked on legal pawn moves reaching the final rank. The UI asks for queen, rook, bishop, or knight before applying the move. Invalid or missing promotion values default to queen inside the engine.

Checkmate and stalemate are computed after each move by checking whether the next player is in check and whether any legal move exists.
